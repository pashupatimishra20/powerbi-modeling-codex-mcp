Who Needs This

  • Power BI modelers who want to automate repetitive model tasks.
  • Analysts who need fast DAX and schema iteration inside Codex.
  • Teams maintaining PBIX/PBIP/TMDL/BIM-based semantic models.
  • Engineers building governed, scripted modeling workflows.

What You Can Do

  • Connect to local Power BI Desktop models.
  • List/edit tables, columns, measures, and relationships.
  • Run DAX validation/execution and model refresh operations.
  • Export/import model artifacts via TMDL or BIM operations.
  • Open PBIR/PBIP projects and create pages, visuals, and field bindings from Codex.
  • Install the whole integration as a Codex plugin with one GitHub command.

What This Adds Over Microsoft's MCP

Microsoft's official powerbi-modeling-mcp gives you the semantic-model tool surface. This repo packages that upstream capability for Codex and adds the missing workflow pieces needed to actually use it comfortably inside Codex sessions.

  • Codex plugin manifest and MCP wiring so the server can be installed and discovered as a Codex plugin.
  • Codex skill instructions that map natural-language requests to the right semantic-model and report-authoring operations.
  • A local powerbi-report-authoring-mcp server for PBIR/PBIP pages, visuals, bookmarks, drillthrough, slicer sync, field-parameter wiring, and interactive controls.
  • Clean install, uninstall, and reinstall scripts for ~/plugins, ~/.agents/plugins/marketplace.json, and ~/.codex/skills.
  • A fallback Node client for listing tools, calling HELP, and troubleshooting outside the Codex UI.
  • GitHub Pages documentation and copy-ready commands for end users.

Install in One Command

For a public GitHub repo, users can bootstrap directly from PowerShell. The installer does a clean reinstall by default, removing any earlier local plugin install, removing any earlier bundled standalone skills mirrored from repo skills/, and deduplicating its marketplace entries before reinstalling:

powershell -NoProfile -ExecutionPolicy Bypass -Command "iex (irm 'https://raw.githubusercontent.com/pashupatimishra20/powerbi-modeling-codex-mcp/main/scripts/bootstrap-install.ps1')"

Compatibility flag

powershell -NoProfile -ExecutionPolicy Bypass -Command "& ([scriptblock]::Create((irm 'https://raw.githubusercontent.com/pashupatimishra20/powerbi-modeling-codex-mcp/main/scripts/bootstrap-install.ps1'))) -Force"

`-Force` is still accepted for older docs, but it is no longer required. Installed paths: `~/plugins/powerbi-modeling-codex`, `~/.agents/plugins/marketplace.json`, and mirrored standalone skills under `~/.codex/skills/` including `powerbi-modeling-mcp` and `powerbi-dashboard-architect`. Then restart Codex desktop.

powershell -ExecutionPolicy Bypass -File .\scripts\install-local.ps1 powershell -ExecutionPolicy Bypass -File .\scripts\uninstall-local.ps1

Usage in Codex

  • Open your PBIX in Power BI Desktop for live semantic-model work, or save it as PBIP/PBIR for report authoring.
  • In Codex, call the skill and connect to the model or open the PBIR project.
  • Run schema/modeling operations or create report pages and visuals in natural language.
  • Validate outputs with list/get operations and ValidateProject after file-based report changes.
USE powerbi-modeling-mcp Connect to 'CVs.pbix' in Power BI Desktop USE powerbi-modeling-mcp list tables USE powerbi-modeling-mcp create a measure to count inactive candidates USE powerbi-modeling-mcp open my PBIR project and create a clustered column chart on a new page

Troubleshooting

  • If MCP handshake fails in-client, use the fallback client in this repo.
  • Confirm local instance discovery (`ListLocalInstances`).
  • Run `Connect` with the returned localhost connection string.
  • Restart Power BI Desktop after external PBIR file edits so the report picks up the changes.
  • Restart Codex after installing or reinstalling so the plugin and bundled standalone skills appear in session context.
  • Use official Microsoft troubleshooting guidance for package/runtime issues.
node .\skills\powerbi-modeling-mcp\scripts\pbi_mcp_client.cjs list-tools node .\skills\powerbi-modeling-mcp\scripts\pbi_mcp_client.cjs help connection_operations

Scope & Boundaries

This integration now covers both semantic model operations and PBIR/PBIP report authoring. Direct binary .pbix editing is still out of scope, so convert PBIX files to PBIP/PBIR when you want Codex to create or update report pages and visuals on disk.

Upstream Foundation

This Codex plugin is an integration layer on top of the official Microsoft project: microsoft/powerbi-modeling-mcp.
