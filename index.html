Microsoft official base

Power BI Modeling MCP server

Microsoft's MCP is the right foundation for semantic-model automation. Their own docs position it around Power BI semantic models and their GitHub repo explicitly states it can only execute modeling operations.

  • Connect to Power BI Desktop, Fabric, and PBIP semantic-model contexts
  • Create and update tables, columns, measures, relationships, translations, and more
  • Validate and execute DAX
  • Work with TMDL and semantic-model project assets
  • Primary documented install path centers on VS Code + GitHub Copilot, with generic MCP manual setup after that

Microsoft also states the modeling server does not modify report pages or other report metadata.

This repo

Codex-ready distribution + PBIR authoring extension

This project takes the official semantic-model MCP and makes it feel native in Codex, then adds a local report-authoring MCP for the PBIR surface that the Microsoft server does not cover.

  • Codex plugin manifest and MCP wiring
  • PowerShell bootstrap install, clean reinstall, and uninstall scripts
  • Standalone skills for discovery and routing in Codex
  • Node fallback client plus generated operation catalog
  • Local PBIR report-authoring MCP for pages, visuals, bookmarks, interactions, mobile layout, and composition

The result is one workflow that spans semantic-model work and file-based report authoring without asking users to stitch it together manually.

End-to-end example

Start in Power BI Desktop, end with a fully authored PBIR report from Codex

  1. 1

    Open a Power BI Desktop .pbit or .pbix and use the official modeling MCP to inspect the semantic model, list tables and columns, analyze data sources, and validate existing DAX.

  2. 2

    Create or update measures, refactor model metadata, check naming consistency, and validate DAX before anything report-facing changes.

  3. 3

    Save the Desktop artifact as PBIP/PBIR when you want file-based report authoring. Open that report project with the local report-authoring MCP.

  4. 4

    Generate or update pages, visuals, bookmarks, bookmark groups, tooltip pages, same-report drillthrough pages, cross-report drillthrough targets, field parameters, synced slicers, and mobile layouts.

  5. 5

    Turn it into a navigable experience with page navigators, bookmark buttons, apply/clear slicer buttons, web URL/Q&A buttons, grouped compositions, visibility states, and layout alignment.

  6. 6

    Run ValidateProject so the final PBIR metadata is schema-valid before reopening the report in Power BI Desktop.

USE powerbi-modeling-mcp connect to 'ExecutiveReview.pbit' in Power BI Desktop
USE powerbi-modeling-mcp inspect the semantic model, list tables/columns, and summarize data sources
USE powerbi-modeling-mcp create or update Net Sales, Margin %, and YoY Growth measures and validate the DAX
USE powerbi-modeling-mcp save the report as PBIP and open the PBIR report project
USE powerbi-modeling-mcp create an Overview page with clustered column, line, donut, card, and slicer visuals
USE powerbi-modeling-mcp create bookmarks for Executive, Finance, and Operations views
USE powerbi-modeling-mcp add a tooltip page, a same-report drillthrough page, and a cross-report drillthrough target
USE powerbi-modeling-mcp create a field parameter for Category, Region, and Month and wire it to a slicer
USE powerbi-modeling-mcp add page navigation, bookmark, back, web URL, and Q&A buttons
USE powerbi-modeling-mcp group the hero controls, align them, set visibility, and bring the group to front
USE powerbi-modeling-mcp generate mobile layout metadata and validate the PBIR project

Install

One command for the plugin, MCP wiring, and bundled skills

Bootstrap install from GitHub

The bootstrap path downloads the repo, runs the local installer, configures the Codex plugin, wires both MCP servers, and mirrors the bundled skills so the setup is ready for Codex sessions. 

powershell -NoProfile -ExecutionPolicy Bypass -Command "iex (irm 'https://raw.githubusercontent.com/pashupatimishra20/powerbi-modeling-codex-mcp/main/scripts/bootstrap-install.ps1')"

Compatibility flag still supported:

powershell -NoProfile -ExecutionPolicy Bypass -Command "& ([scriptblock]::Create((irm 'https://raw.githubusercontent.com/pashupatimishra20/powerbi-modeling-codex-mcp/main/scripts/bootstrap-install.ps1'))) -Force"

Local script path

If you cloned the repo already, use the local scripts directly:

powershell -ExecutionPolicy Bypass -File .\scripts\install-local.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\uninstall-local.ps1

After install, restart Codex Desktop if you use the desktop app. For Codex CLI, open a new session so the plugin and skills load cleanly.
